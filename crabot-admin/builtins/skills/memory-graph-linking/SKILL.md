# 记忆图谱建链

给长期记忆条目之间建立**有类型的关联链接**，让孤立条目长成知识图谱，供召回时沿链补全上下文。

> 调用方（「重建记忆图谱」任务 / 每日反思第五步 b）会指定**处理范围**（全部 confirmed / 仅近期新增）与**是否覆盖式**（重建 = 先清旧再建；反思 = 增量、不清）。本技能只定义「一条链该不该建、建成什么关系」——这部分两条路径完全一致。

## 建链流程

对范围内的每条目标条目 N：

1. **找候选**：`mcp__crab-memory__search_long_term({ query: <N 的 brief>, filters: { status: "confirmed" }, k: 5 })`。
2. **判定（默认不连）**：逐个候选判断 N 与候选间**是否确有【具体、有信息量】的关系**（见下方判据）。
3. **写入**：确有关系时 `mcp__crab-memory__set_memory_links({ id: <N 的 id>, links: [{ target, relation }] })`。
   - 取数用 `mcp__crab-memory__list_entries({ status: "confirmed", limit, offset })` 翻页（全量）或 `list_recent`（近期）。

## 判据（惜墨如金，目标是稀疏有信息量的图，不是毛球）

**默认不连。绝大多数条目对之间不该有边。** 只有确有具体关系才连。

**relation 受控词表（只能取这 4 个，按优先级从上到下）：**

- `refines`：A 细化 / 深化 B（同主题下 A 更具体、更深入）。该用：A 是 B 的进阶说明 / 更细颗粒版本。
- `depends_on`：A 以 B 为前提、依赖 B 才成立。该用：不先满足 B，A 的结论就不成立。
- `part_of`：A 是 B 的组成部分。该用：A 是 B 这个更大整体里的一块。
- `related`：泛相关**兜底**——仅留给【真有意义、非显而易见】的跨条目引用。能用上面三种更具体的就别退回 related。

**硬性禁令：**

- **【严禁】因为"同项目 / 同主题 / 话题接近"就连 `related`** —— 同主题由实体/标签聚类表达，不是建链的职责。`search_long_term` 找到的相似条目**多数只是"同主题"**，相似 ≠ 该连；拿不准就**不连**。
- **`related` 是对称关系：A↔B 只建一条（A→B 或 B→A 任选其一），【严禁】正反各建一条** —— 双向重复纯属噪音。`refines` / `depends_on` / `part_of` 有方向，按"A 对 B"的真实方向建。
- **不与已有结构关系重复造边**：`source_cases`（rule→case）、`invalidated_by`（新→旧）已表达的关系不要再用 link 重复。
- **近重复 ≠ 建链**：若两条几乎是同一条经验，那是**去重合并**的事（交给 PE-gated / Case→Rule），不要用 related 把它们连起来。

## 关系判定速查

| 看到这种 | 用 |
|---|---|
| A 是 B 的更细颗粒 / 进阶说明 | `refines` |
| 不先满足 B，A 的结论不成立 | `depends_on` |
| A 是 B 这个更大整体里的一块 | `part_of` |
| 确有跨条目引用价值、但说不出上面三种结构 | `related`（慎用、且对称只建一条） |
| 只是同项目 / 同主题 / 话题接近 | **不连** |
| 几乎是同一条经验 | **不连**（该去重合并） |
